推荐方案

综合信息密度、实现成本、与现有 UI 风格匹配度，推荐以下两个方案择一实施：

首选 · 信息优先

方案 A · 容量胶囊徽标

理由：任务数是用户最关心的信息（"今天有多少事"），胶囊直接显示数字 + 颜色等级，扫视即知。实现最简单（1 个 div + class 绑定），与现有 NTag 风格一致。适合需要快速浏览整月负载的场景。

建议尺寸：min-w-[22px] h-[20px] px-2 rounded-full text-[11px] font-bold，与今日小蓝点错位（徽标右上、蓝点居中下）。

备选 · 专业感优先

方案 D · 圆环仪表盘

理由：圆环 + 中心数字同时表达"百分比 + 精确数"，视觉最精致，与 SelfPilot 工具型定位契合。SVG stroke-dasharray 实现弧长动画，hover 时可加旋转过渡。适合强调"容量监控"的仪表盘式体验。

建议尺寸：26×26px，stroke-width 3px，半径 10px，circumference ≈ 62.83。

💡 所有方案均保留背景 tint（bg-green-50/amber-50/red-50/purple-50）作为辅助视觉通道，并与现有 getLoadOfDay() tooltip 配合，hover 时显示"3 个任务（目标A×2, 目标B×1）"明细。选定方案后可在 CalendarView.vue 中替换 <!-- P2-5：负载指示条 --> 块（约 L1021-1035）。
