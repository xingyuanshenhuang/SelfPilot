# 数据管理系统重设计 — 实现计划

## Context

当前 SettingsView.vue 的「数据管理」区域把 4 个操作（JSON导出、JSON导入、一键备份、恢复备份）平铺排列，缺乏视觉分组和方式对比。用户无法直观理解两种备份方式的区别，也不清楚该选哪个。需要重新设计 UI，清晰区分两种备份方式，并提供引导性的对比信息。

## 设计方案

### 核心：双卡片分区 + 对比表格

将原来的单个「数据管理」卡片拆分为 **三个卡片**：

1. **「一键备份」卡片**（推荐，放在前面）— SQLite 原生备份/恢复
2. **「数据导入导出」卡片** — JSON 格式导入/导出
3. **「备份方式对比」卡片** — 对比表格，帮助用户选择

### 卡片 1：一键备份（推荐）

```
┌─ 🗄️ 一键备份（推荐）─────────────────────────┐
│                                                │
│  一键备份所有数据，恢复时完整还原，速度快。    │
│  适合日常备份使用。                            │
│                                                │
│  [💾 立即备份]  [📂 恢复备份]                  │
│                                                │
│  💡 恢复时会自动备份当前数据，不用担心丢失。   │
└────────────────────────────────────────────────┘
```

- 标题旁加 `NTag` 标注「推荐」
- 底部加提示文字：恢复时的安全网机制
- 两个按钮并排，主按钮「立即备份」

### 卡片 2：数据导入导出

```
┌─ 📋 数据导入导出 ─────────────────────────────┐
│                                                │
│  以 JSON 格式导出或导入数据。适合跨版本迁移    │
│  或选择性合并数据。                            │
│                                                │
│  [📤 导出数据]   [📥 导入数据]                 │
│                                                │
│  导入时可选择冲突处理方式：                    │
│  · 跳过冲突项（保留本地）                      │
│  · 覆盖冲突项（使用导入数据）                  │
│  · 重命名导入项（保留双方）                    │
└────────────────────────────────────────────────┘
```

- 说明文案强调「跨版本迁移」和「选择性合并」
- 导入按钮点击后弹出原有的导入确认弹窗（含冲突模式选择）
- 将冲突模式说明从弹窗移到卡片内，提前告知用户

### 卡片 3：备份方式对比

使用 `NDescriptions` 对比表格：

```
┌─ ❓ 两种备份方式有什么区别？ ─────────────────┐
│                                                │
│  ┌──────────┬──────────────┬──────────────┐   │
│  │          │ 一键备份      │ 数据导入导出  │   │
│  ├──────────┼──────────────┼──────────────┤   │
│  │ 备份范围 │ 全部数据     │ 全部数据      │   │
│  │ 数据格式 │ 数据库文件    │ JSON 文本     │   │
│  │ 恢复方式 │ 替换全部数据  │ 可选择合并    │   │
│  │ 恢复速度 │ 快（秒级）   │ 较慢          │   │
│  │ 适用场景 │ 日常备份恢复  │ 跨版本迁移    │   │
│  │ 推荐度   │ ⭐⭐⭐      │ ⭐⭐         │   │
│  └──────────┴──────────────┴──────────────┘   │
└────────────────────────────────────────────────┘
```

使用 naive-ui 的基础 HTML table（因为 NDataTable 对这种简单对比太重），加上 Tailwind 样式美化。

### 导入确认弹窗改进

现有的导入弹窗保留，但增加 JSON 预览信息（文件版本、导出时间、数据量统计），让用户确认导入内容正确。

在 `handleFileChange` 读取文件后，尝试解析 JSON 并提取摘要信息（version、exported_at、goals/tasks/encouragements 数量），显示在确认弹窗中。

## 实现步骤

### Step 1: 重写 SettingsView.vue 的数据管理区域

**Script 部分新增**：

- `importPreviewInfo` ref：存储解析后的 JSON 摘要信息
- `parseExportSummary(jsonStr)` 函数：解析 JSON 并提取 version、exported_at、各数据数量

**Template 部分重写**：

- 删除原来的单个「数据管理」NCard
- 替换为三个 NCard：一键备份、数据导入导出、备份方式对比
- 对比表格使用原生 `<table>` + Tailwind 样式

### Step 2: 改进导入确认弹窗

- 在 `handleFileChange` 中解析 JSON 并提取摘要
- 弹窗中显示：备份版本、导出时间、数据量（X 个目标、Y 个任务、Z 个鼓励语）
- 保留冲突模式选择

### Step 3: 构建验证

- `npx vue-tsc --noEmit`
- `cargo check`（无需改动后端）
