## 三、P1 — 扩展任务模型

### P1-1 周期性任务补全

**现状**:[services/split_service.rs#L250](file:///workspace/src-tauri/src/services/split_service.rs#L250-L280) `split_repeat_tasks` 只支持每日重复,PRD §4.2 承诺的「每周几/每月几号」未实现。

**目标**:支持完整的周期规则。

**数据模型扩展**:

```rust
// RepeatSplitInput 扩展
pub struct RepeatSplitInput {
    pub goal_id: String,
    pub name: String,
    pub start_date: String,
    pub end_date: Option<String>,
    pub plan_qty: Option<f64>,
```

**涉及文件**:

**验收标准**:

- 选「每月1号15号」+ 日期范围 → 生成每月1号和15号的任务

### P1-2 任务依赖关系

**现状**:`tasks` 表无依赖字段,无法表达「B 必须在 A 完成后执行」。

**目标**:支持任务间依赖,影响排序和展示。

**数据模型扩展**:

```sql
-- 新增依赖表(多对多)
CREATE TABLE task_dependencies (
    task_id TEXT NOT NULL,           -- 当前任务
    depends_on_task_id TEXT NOT NULL, -- 依赖的任务
    PRIMARY KEY (task_id, depends_on_task_id),
    FOREIGN KEY (task_id) REFERENCES tasks(id) ON DELETE CASCADE,
    FOREIGN KEY (depends_on_task_id) REFERENCES tasks(id) ON DELETE CASCADE
);
```

**功能点**:
- 任务详情弹窗增加「依赖任务」多选(同目标下其他任务)
- 今日待办:依赖未完成的任务标灰,不可勾选完成
- 日历视图:依赖未满足的任务显示锁图标
- 重新规划:依赖链决定排序(被依赖的任务排在前面)
- 循环依赖检测:添加依赖时校验无环

**涉及文件**:
- 后端:新增 [commands/task.rs](file:///workspace/src-tauri/src/commands/task.rs) `add_dependency` / `remove_dependency` / `list_dependencies`
- 迁移:新增 `005_task_dependencies.sql`
- 前端:[TaskItem.vue](file:///workspace/src/components/TaskItem.vue)、[DashboardView.vue](file:///workspace/src/views/DashboardView.vue)、[CalendarView.vue](file:///workspace/src/views/CalendarView.vue)

**验收标准**:
- 任务 B 依赖任务 A,A 未完成 → B 在今日待办显示为灰色不可点
- A 完成后 → B 自动变为可点
- 尝试设置 B 依赖 A 且 A 依赖 B → 拦截提示循环依赖

### P1-4 时间预算模型

**现状**:任务只有 `plan_qty`(数量),无法按时间拆解。


- 今日待办显示「今日总时长」汇总

**涉及文件**:

## 八、关键决策点

### 决策 1:任务依赖是否做?

**背景**:P1-2 是把 SelfPilot 从「待办工具」升级为「规划系统」的关键,但实现复杂度高。

**建议**:**做**。这是产品差异化的核心,且数据模型清晰(多对多依赖表),主要工作在前端交互。

### 决策 2:云同步何时启动?

**背景**:P3-1 工程量极大,涉及服务端开发。

**建议**:**延后到有稳定用户群后**。当前离线优先已满足核心场景,云同步是「锦上添花」而非「雪中送炭」。

### 决策 3:是否引入 AI?

**背景**:P3-4 是热门方向,但依赖外部 API 或本地模型。

**建议**:**作为可选增值功能**。先做规则引擎(基于历史数据的简单建议),验证用户需求后再考虑接 LLM。
