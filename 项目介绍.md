> 版本：0.1.0  
> 检查日期：2026-06-25  
> 对应文档：PRD v1.0、分阶段交付计划、技术栈详情

## 一、项目背景

**SelfPilot** 是一款面向自学者、学生及考证人群的离线个人任务规划桌面应用。它解决了传统 Todo 工具在“长期目标管理”上的不足：用户常常只能记录零散任务，却难以把一个宏大目标（如“30 天背完 3000 个单词”“3 个月通过 PMP”）自动拆解为可执行的每日任务，并实时掌握整体进度。

SelfPilot 的核心价值在于：

- **目标 → 阶段 → 任务** 三级结构，兼顾宏观规划与微观执行；
- **自动拆解算法**，按截止日期把目标总量平均分配到每天；
- **进度自动向上汇总**，子任务完成情况实时反馈到阶段与目标；
- **离线优先**，数据完全存储在本地 SQLite，无需登录、无需网络。

## 二、核心功能

应用围绕 6 个固定模块展开，对应左侧导航栏：

| 模块 | 功能说明 |
|------|----------|
| **目标总览** | 展示所有目标完成百分比（环形进度）、今日待办、逾期任务，并支持快速完成 / 跳过 / 补完成。 |
| **目标树** | 以层级卡片形式管理目标、阶段与任务；支持创建目标、自动拆解、添加阶段、重新规划。 |
| **日历视图** | 月 / 周 / 日三视图展示每日任务分布，支持点击查看详情并批量完成 / 跳过。 |
| **数据统计** | 柱状图（各目标完成度）、折线图（近 7 / 30 天趋势）、日历热力图、完成预测。 |
| **鼓励语库** | 预设 + 自定义鼓励语，按连续打卡天数智能抽取等级，全部目标完成时触发庆祝语。 |
| **设置** | 浅色 / 深色主题切换，JSON 数据导出 / 导入，支持跳过 / 覆盖 / 重命名三种冲突处理。 |

### 3.3 数据模型

采用 **UUID + parent_id + path** 的三级树形结构：

- `Goal`（目标，一级节点）
- `Stage`（阶段，二级节点，可选）
- `Task`（任务，三级节点，实际执行单元）

任务状态：`pending`（未完成）、`partial`（部分完成）、`done`（已完成）、`skipped`（已跳过）。

## 五、已实现的关键特性

### 5.1 自动拆解与重新规划

- 按截止日期剩余天数（不含今天，且 ≥ 1）将目标总量平均分配，余数自动放到前几天。
- 每个任务记录计划数量与单位，支持数量型与布尔型（plan_qty = 1）任务。
- 重新规划时过滤已跳过任务，保留 `is_manual = 1` 的手动调整项，并展示 Before / After 预览。

### 5.2 任务生命周期

- 完成：支持部分完成（actual_qty < plan_qty → partial）。
- 跳过：标记为 skipped，不计入完成，不影响后续计划。
- 补完成：仅更新历史任务 actual_qty 与状态，**绝不重新分配未来任务**。
- 批量操作：日历日视图支持多选后批量完成 / 跳过。

### 5.3 进度汇总

- 目标进度 = 子任务实际完成量 / 子任务计划量（跳过任务不计入）。
- 阶段进度通过 SQL 聚合实时计算，随任务状态变化自动刷新。

### 5.4 数据统计与预测

- 近 7 / 30 天完成趋势折线图。
- 各目标完成百分比柱状图。
- 日历热力图（90 / 180 / 365 天切换）。
- 基于过去 7 天平均速度的完成预测，输出 `ahead` / `on_track` / `need_speed` / `completed` / `no_data` / `no_deadline` 等状态与建议。

### 5.5 鼓励机制

- 预设 10+ 条鼓励语，并分 normal / advanced / highlight / celebration 四级。
- 完成当日首个任务后弹出鼓励语弹窗；全部目标完成时触发庆祝语。
- 连续打卡规则严格遵循 PRD：每天至少完成 1 个任务计 1 天；当天无任务不中断也不计入；有任务未完成则中断。

### 5.6 数据安全

- 本地 SQLite 存储于应用数据目录（`selfpilot.db`）。
- JSON 全量导出 / 导入，支持 skip / overwrite / rename 三种 ID 冲突策略。
- sqlx 迁移脚本管理数据库版本（001_init → 002_encouragement_settings → 003_encouragement_level）。

## 六、后续发展规划

### 6.1 近期补齐（建议优先级 P0 ~ P1）

1. **目标树手动添加任务入口**：在目标或阶段卡片上增加“添加任务”按钮，调用已有的 `create_task` Command。
2. **任务编辑与拖拽归属**：
   - 任务右键 / 编辑按钮：修改名称、日期、计划数量；
   - 拖拽任务到阶段，或下拉选择阶段归属（`move_task` 已存在）。
3. **鼓励语编辑**：在鼓励语库中为自定义鼓励语提供修改入口。
4. **清理未使用依赖**：`@tauri-apps/plugin-fs` 与 `@tauri-apps/plugin-dialog` 若确认不使用，可从 package.json 移除。
5. **构建环境修复**：排查 Windows 下 Tauri 构建脚本的 `rc.exe` / 进程退出码 0 问题，确保 `cargo check` 与 `tauri build` 可稳定运行。

### 6.2 中期增强（P1 ~ P2）

1. **周期性手动任务**：支持“每日 / 每周几 / 每月几号”重复规则（PRD 中已提及，当前未实现）。
2. **目标编辑**：修改目标名称、截止日期、总量；总量变化时给出重新拆解提示。
3. **数据导入 SQLite 原生备份**：除 JSON 外，提供 `.db` 文件备份选项。
4. ** richer 异常提示**： deadline 为空 / 已过期时的前置拦截已在后端，可补充前端表单校验。

### 6.3 远期展望（P2 / 预留）

1. **云端同步**：技术栈文档中预留的 `sync.rs` / `sync_service.rs` 可接入 HTTP API。
2. **更大数据量优化**：当目标树节点超过 1000 时，考虑引入虚拟滚动或分页加载。
3. **多语言与国际化**：当前为中文界面，可扩展 i18n。

## 七、总结

SelfPilot 已完成 PRD 中 6 个核心模块的主体功能，Sprint 1 / 3 / 4 / 5 / 6 基本交付完毕，Sprint 2 的目标树交互仍有小幅收尾空间。技术栈整体与文档规划一致，仅在 SQLite 驱动、树形组件、日历组件、导入导出实现方式上存在“功能等价但选型不同”的差异。

当前项目已具备**可用 Demo / 内测版本**的条件，建议优先补齐目标树内手动添加任务与任务拖拽归属两个交互，随后进行一次完整的前后端联调与打包验证。
